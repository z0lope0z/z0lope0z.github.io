What makes HAProxy Highly Available?

Plot twist: HA isn't built in to HAProxy

HAProxy is a popular load balancer and proxy server. But what makes it highly available? How does it work internally? After researching on this topic, it's highly available because it utilizes a daemon called keepalived. Keepalived enables two haproxy instances to share a virtual ip address with another passive haproxy instance. The virtual ip is assigned to the active haproxy process. When the active haproxy process fails, the virtual ip gets assigned to the passive (now active) haproxy process. Keepalived, in turn uses a combination of Virtual Router Redundancy Protocol and Bi-directional Forwarding Protocol to quickly check if the associated haproxy instance is alive, and reassign the virtual ip. HAProxy doesn't have to use keepalived, it can also be ucarp, or any other daemon/tool that supports ip failovers. references: https://www.researchgate.net/publication/327402582_IoT-F2CDM-LB_IoT_Based_Fog-to-Cloud_and_Data-in-Motion_Architectures_with_Load_Balancing http://www.formilux.org/archives/haproxy/1003/3259.html
