Monitoring s3 objects with AWS Lambda

ah lambda, my best friend!

A data pipeline is currently writing logs in s3. I needed to monitor these files. The target metric's value is derived from the file's metadata.


Luckily, there's an existing cloudwatch to datadog integration. I just needed to publish the metric in cloudwatch and datadog automatically picks it up.


Architecture


1. There are two triggers for this lambda function, (1) scheduled and (2) s3 event based
2. Whenever there is a trigger, the lambda function queries the s3 bucket with a business logic that converts s3 meta data to a metric
3. The metric is emitted to cloudwatch
